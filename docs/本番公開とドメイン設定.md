# 本番公開とドメイン設定（japan-clarinet-association.jp）

このドキュメントでは、**japan-clarinet-association.jp** でサイトを公開する手順をまとめています。  
ホスティングは **Vercel**（Next.js 推奨）を前提にしています。

---

## 全体の流れ

1. **Vercel にプロジェクトをデプロイする**（まだの場合）
2. **Vercel でカスタムドメインを追加する**
3. **ドメイン取得先（レジストラ）で DNS を設定する**
4. **環境変数 `NEXT_PUBLIC_SITE_URL` を本番用に設定する**

---

## 1. Vercel にデプロイする

### 1-1. Vercel アカウント

- [vercel.com](https://vercel.com) でアカウントを作成（GitHub 連携がおすすめ）
- まだリポジトリを GitHub に上げていない場合は、プッシュしておく

### 1-2. 新規プロジェクトとしてインポート

1. Vercel ダッシュボードで **Add New… → Project**
2. **Import Git Repository** で、このサイトのリポジトリを選択
3. **Framework Preset**: Next.js が自動検出されます
4. **Root Directory** がリポジトリルートで問題なければそのまま
5. **Environment Variables** で、本番用の環境変数を設定（後述の「4. 環境変数」を参照）
6. **Deploy** を実行

初回デプロイ後、`xxxx.vercel.app` のような URL で公開されます。

### 1-3. 今後の更新

- **Git の main ブランチに push** すると、Vercel が自動で再デプロイします
- 本番用ブランチ名が main 以外の場合は、Vercel の **Settings → Git** で本番ブランチを指定してください

---

## 2. Vercel でカスタムドメインを追加する

1. Vercel ダッシュボードで **対象プロジェクト** を開く
2. **Settings → Domains** を開く
3. **Add** に **japan-clarinet-association.jp** を入力して追加
4. （任意）**www.japan-clarinet-association.jp** も追加する場合  
   → 推奨: どちらか一方を「メイン」にし、もう一方をリダイレクト用にするとよいです（後述）

追加直後は「設定待ち」になり、**次のステップで DNS を設定すると有効**になります。

---

## 3. ドメイン取得先（レジストラ）で DNS を設定する

**japan-clarinet-association.jp** を取得したサービス（お名前.com、ムームードメイン、Google Domains、Cloudflare など）の管理画面で、次のように設定します。

### 3-1. Vercel が案内する値の確認

Vercel の **Settings → Domains** で **japan-clarinet-association.jp** をクリックすると、

- **A レコード** 用の IP アドレス
- **CNAME** 用のホスト名（例: `cname.vercel-dns.com`）

が表示されます。表示に従ってください（Vercel の仕様変更があるため、ここでは一般的な例のみ記載します）。

### 3-2. 設定例（ルートドメイン japan-clarinet-association.jp）

| タイプ | ホスト名（名前） | 値（参照先） |
|--------|------------------|--------------|
| **A**  | `@` または空     | Vercel に表示されている IP（例: 76.76.21.21） |
| **CNAME**（代替） | `@` または空 | 使えないレジストラが多いため、通常は A レコードを使用 |

- 多くのレジストラでは **ルートドメイン（apex）には A レコード** を設定します。
- **CNAME フラットニング** をサポートしている場合は、Vercel の案内どおり CNAME を指定することもあります。

### 3-3. www を使う場合（任意）

**www.japan-clarinet-association.jp** も Vercel に追加している場合:

| タイプ   | ホスト名（名前） | 値 |
|----------|------------------|-----|
| **CNAME** | `www`            | `cname.vercel-dns.com`（Vercel に表示されている値） |

- レジストラによっては「CNAME の値」が `cname.vercel-dns.com.` のように末尾にドット付きで案内されることがあります。その場合はそのとおり入力してください。

### 3-4. 反映時間

- DNS の反映には **数分〜最大 48 時間** かかることがあります。
- Vercel の Domains 画面で「有効」「検証済み」になれば本番ドメインでアクセスできます。

---

## 4. 環境変数（本番用）

本番で **リダイレクト・メールリンク・決済後の戻り先** などを正しく動かすため、Vercel で次を設定してください。

1. プロジェクトの **Settings → Environment Variables**
2. 次の変数を **Production** 用に追加（既にある場合は編集）:

| 変数名 | 値（例） | 説明 |
|--------|----------|------|
| `NEXT_PUBLIC_SITE_URL` | `https://japan-clarinet-association.jp` | サイトの本番URL（末尾の `/` なし） |

- **www をメインにする場合** は  
  `NEXT_PUBLIC_SITE_URL=https://www.japan-clarinet-association.jp` にします。
- 変更後は **再デプロイ**（Deployments から最新を Redeploy するか、main に push）すると反映されます。

### ほかの環境変数も Vercel に登録する

このサイトは、**Supabase（会員DB・ログイン）**・**Stripe（決済）**・**メール送信（Gmail）** を使うため、それ用の「環境変数」も Vercel に登録する必要があります。

- **環境変数** = サーバー側で参照する「設定の名前と値」のペア（例: `NEXT_PUBLIC_SUPABASE_URL` = `https://xxxx.supabase.co`）
- **Production** = 本番サイト（japan-clarinet-association.jp）用の設定。Vercel の「Environment Variables」で **Production** にチェックを付けて登録すると、本番デプロイ時に使われます。
- **本番用の値** = 開発用のテスト値（`sk_test_xxxx` など）ではなく、本番用の Supabase プロジェクトの URL／本番用の Stripe キー／実際に送信に使う Gmail アドレスなど、公開サイトで使う本当の値です。

登録する変数名の一覧は、プロジェクト直下の **`.env.example`** に書いてあります。  
Vercel の **Settings → Environment Variables** で、その名前と同じ変数をひとつずつ追加し、本番用の値（Supabase ダッシュボードや Stripe ダッシュボードで確認できる本番の URL・キーなど）を入力してください。

---

## 5. www のリダイレクト（任意）

- **japan-clarinet-association.jp** をメインにしたい場合  
  → Vercel で **www.japan-clarinet-association.jp** を追加し、Vercel の「Redirect」で  
  `www.japan-clarinet-association.jp` → `japan-clarinet-association.jp` にリダイレクトすると、どちらでアクセスしてもルートドメインに統一できます。
- **www をメイン** にしたい場合は、その逆（ルート → www）を設定できます。  
  いずれも **Settings → Domains** でドメインを追加したあと、**Redirects** で設定可能です。

---

## 6. 確認チェックリスト

- [ ] Vercel にデプロイされ、`xxxx.vercel.app` で表示できる
- [ ] Vercel の Domains に **japan-clarinet-association.jp** を追加した
- [ ] レジストラで **A レコード**（必要なら CNAME）を設定した
- [ ] Vercel の Domains で japan-clarinet-association.jp が「検証済み」になっている
- [ ] ブラウザで **https://japan-clarinet-association.jp** にアクセスできる
- [ ] Vercel の Environment Variables で **NEXT_PUBLIC_SITE_URL** を  
  `https://japan-clarinet-association.jp` に設定し、再デプロイした
- [ ] ログイン・決済・メール内リンクなど、サイト内のリンクが本番URLで動くことを確認した

---

## トラブルシュート

| 現象 | 確認すること |
|------|----------------------|
| ドメインでアクセスできない | DNS の反映待ち（最大 48 時間）。レジストラの DNS 設定が正しいか再確認。 |
| 「証明書エラー」「安全ではない」 | Vercel が SSL を自動発行するため、検証が完了するまで数分〜数十分かかることがある。Domains の状態を確認。 |
| ログイン後に飛ぶURLがおかしい | `NEXT_PUBLIC_SITE_URL` が本番URLになっているか、Production 用に設定されているか確認。 |
| **マイページで「fetch failed」** | 本番のみでログインできない場合。Vercel の **Environment Variables** で **Production** に `NEXT_PUBLIC_SUPABASE_URL` と `NEXT_PUBLIC_SUPABASE_ANON_KEY` が Supabase の Project URL・anon key と一致して設定されているか確認。変更後は **Redeploy** 必須。Supabase のプロジェクトが Paused になっていないか、カスタムドメインの場合は Project URL が有効か確認。 |
| デプロイが失敗する | Vercel の **Deployments → 該当ログ** を確認。`npm run build` がローカルで通るか確認。 |

必要に応じて、Vercel の [ドメイン公式ドキュメント](https://vercel.com/docs/concepts/projects/domains) も参照してください。
