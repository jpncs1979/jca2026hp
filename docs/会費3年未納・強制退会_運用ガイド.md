# 会費3年未納・強制退会の運用ガイド（初心者向け）

この機能で **Supabase のダッシュボードで難しい設定を新たに行う必要はありません。**  
必要なのは次の2点だけです。

1. **データベースの更新（マイグレーション）** … いつもどおり SQL を当てる  
2. **「3年未納チェック」を定期実行する仕組み** … サイトを置いているサービス（多くは Vercel）の **環境変数** だけ触る（Supabase ではない）

---

## 1. データベースの更新（マイグレーション）

`supabase/migrations/020_profiles_expelled_three_year.sql` の内容をデータベースに反映します。

### 方法A: Supabase の SQL Editor（よくあるやり方）

1. [Supabase](https://supabase.com) にログインし、対象プロジェクトを開く  
2. 左メニュー **「SQL Editor」** を開く  
3. **「New query」** で新しいクエリを作る  
4. ファイル `020_profiles_expelled_three_year.sql` の **中身をすべてコピー** して貼り付ける  
5. **「Run」** を押す  
6. 緑の成功メッセージが出れば完了です  

※ すでに CLI や GitHub でマイグレーションを流している運用なら、**いつもと同じ方法**でこのファイルだけ追加して実行して構いません。

### 失敗したとき

- `constraint` や `already exists` などのエラーが出た場合は、**二重実行**や**古い制約名**の可能性があります。そのときは開発担当または Supabase のサポートログを確認してください。

---

## 2. 「3年未納チェック」とは何か

- **毎日1回**（設定どおり）、サーバー上のプログラムが  
  「有効会員のうち、**事業年度ベースで直近3年連続で会費が未納**の人はいるか」を調べます。  
- 該当者がいれば、その人のステータスを **「強制退会（expelled）」** に自動で変えます。  
- **メール送信や Supabase 内のトリガー設定は使いません。** 普通の API を叩くだけです。

---

## 3. 定期実行の設定（Supabase ではなく「サイトのホスト」側）

このチェック用の URL は次のとおりです（ドメインは実際のサイトに読み替えてください）。

```text
https://（あなたのサイトのドメイン）/api/cron/membership-three-year-expulsion
```

この URL を呼ぶとき、**秘密の合言葉（`CRON_SECRET`）** が一致しないと動きません（いたずら防止用です）。

### 3-A. Vercel でサイトを公開している場合（よくある）

1. Vercel のダッシュボードで **該当プロジェクト** を開く  
2. **Settings** → **Environment Variables** を開く  
3. **Name** に `CRON_SECRET`、**Value** に **英数字の長めの文字列**（自分で決めてよい。例: パスワード生成サイトで32文字など）を入れて保存  
4. **Production（本番）** にチェックを入れて保存（Preview も使うなら同様に追加可）  
5. **Deployments** から **Redeploy** して、環境変数を反映させる  

`vercel.json` に Cron の行を入れてある場合、Vercel が **自動で上記 URL を呼びます。**  
そのとき、Vercel は `Authorization: Bearer （CRON_SECRETの値）` を付けるので、**同じ値を Vercel の環境変数に入れておけば動きます。**

**Supabase 側で Cron や Edge Functions を設定する必要はありません。**

### 3-B. Vercel を使っていない場合

次のどれかで **1日1回程度**、上記 URL を呼び出してください。

- サーバー管理会社の「Cron」機能  
- 無料の外部サービス（例: 「cron job」「定期HTTPリクエスト」で検索）で、  
  - URL: 上記  
  - メソッド: **GET**  
  - ヘッダー: `Authorization` = `Bearer （あなたが決めたCRON_SECRET）`  
    または `x-cron-secret` = **（同じ CRON_SECRET の値）**  

Next.js を動かしているサーバーの **環境変数** にも、**同じ `CRON_SECRET`** を設定してください（コードがそれと照合します）。

---

## 4. 事務局での「再入会を許可するとき」

強制退会になった方が、**同じメールアドレスで** もう一度ウェブ入会したい場合:

1. 管理画面の **会員詳細** を開く  
2. **「再入会のためメールを退避（ウェブ入会を許可）」** ボタンを押し、確認に答える  
3. システムがログイン用メールを退避用アドレスに差し替え、ステータスを **期限切れ** にします  
4. 本人には **同じメールアドレスで** `/membership/join` から申し込んでもらう  

**Supabase で手作業する作業はありません。** 管理画面だけで完結します。

---

## 5. よくある質問

### Q. Supabase で「Webhooks」や「Database の Cron」は必要？

**不要です。** この機能は Next.js の API とマイグレーションだけで動きます。

### Q. `SERVICE_ROLE_KEY` や `NEXT_PUBLIC_SUPABASE_URL` は？

サイトがもともと動いているなら **すでに設定済み** です。追加の Supabase 設定は不要です。

### Q. 3年未納チェックを止めたい

- Vercel の場合: `vercel.json` の該当 Cron 行を削除するか、Vercel 側の Cron をオフにする  
- 外部 Cron の場合: そのサービスでスケジュールを止める  

---

## 6. 通常会員の「退会」（編集画面）

会員情報の編集画面にある **「退会（資格喪失）」** は、**プロフィールや入金履歴を消しません。**  
有効会員としての資格だけを止め（ステータスを期限切れにし、ログイン用アカウントがあれば削除）、データはデータベースに残ります。

---

## 7. まとめ

| やること | どこでやるか |
|---------|-------------|
| テーブル・ステータス列の追加 | **マイグレーション（SQL）** … Supabase の SQL Editor でも可 |
| 毎日の未納チェック | **Vercel 等の環境変数 `CRON_SECRET` + Cron** … Supabase ではない |
| 再入会の許可 | **管理画面のボタン** |
| 任意の退会（データ保持） | **会員編集の「退会（資格喪失）」** |

Supabase の「難しい設定」は前提にしていません。マイグレーションと、サイトをホストしている場所の環境変数・定期実行だけ意識してください。
