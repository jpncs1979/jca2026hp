# 協会入会・自動継続決済システム 実装メモ

## 1. 決済・更新ロジック (Stripe)

### 事業年度（会費）

- **2月1日〜翌年1月31日**を「N年度」（開始年 N）とする（`src/lib/membership-fiscal-year.ts`）。会費の表示・未納判定・1月の自動請求の対象年度はこれに基づく。

### 会員資格（会員年度）

- **4月1日〜翌年3月31日**が会員としての資格期間。`memberships.expiry_date` にはその期間の**末日（多くは 3/31）**を入れる。
- 事業年度の会費の対象が「開始年」の最大を `maxPaid` とすると、資格の末日は **`membershipEligibilityEndIsoFromMaxPaidBusinessFiscalYear(maxPaid)` = `(maxPaid+1)-03-31`**（例: 事業2026年度分のみ支払済 → `2027-03-31` ＝ 2026/4/1〜2027/3/31 の会員年度）。
- **1月までに翌事業年度分の会費が入金**された場合は、その**次の4月1日**から始まる会員年度に更新される（実装上は上記の式で `expiry_date` を延ばす）。

### 入会時決済

- `POST /api/membership/checkout` で入会フォーム送信 → Stripe Checkout（`mode: payment`）。`customer_creation: always` と `payment_intent_data.setup_future_usage: off_session` で **Stripe Customer と将来のオフセッション課金用の支払方法**を保存。
- **2〜10月入会**: 入会金＋**当該事業年度**の会費（`getMembershipJoinAmount` / `joinMembershipFeeFiscalYears`）。
- **11・12・1月入会**: 入会金＋**翌事業年度**の会費のみ。
- 決済完了は Webhook `checkout.session.completed`（`metadata.type=membership_join`）で profile・memberships・payments 作成、**会員資格の末日**は上記ルール、入会完了メール送信。`profiles.stripe_customer_id` を保存し、Customer の default payment method を同期。

### マイページ年会費

- `POST /api/mypage/membership-renew/checkout`：未納年度の Stripe Checkout。既存 `stripe_customer_id` があれば同一 Customer でセッション作成。

### 毎年1月のカード自動引き落とし

- **`GET /api/cron/membership-january-charge`**（`CRON_SECRET` 必須）。
- **JST 1月22日のみ**実行（Vercel Cron: `0 15 21 1 *` = 1/21 15:00 UTC ≒ 1/22 0:00 JST）。Stripe の入金が概ね T+2 営業日となるため、1/31 までの着金余裕を見て 22 日に設定（`src/lib/membership-january-charge.ts`）。
- 対象: `status=active`、**`is_css_user` が true でない**（銀行振込 CSS は除外）、`stripe_customer_id` あり、かつ対象年度が未納。
- 徴収する年度: **その年の2月から始まる事業年度**（1月中は `fiscalYearForDate(JST)+1`）。
- 実装: Stripe PaymentIntent `off_session` + `confirm`。成功後 `payments` に `metadata.january_auto=1` を付与し、`memberships.expiry_date` を更新。

### 銀行振込（CSS）

- **`is_css_user=true`**: 別サービス（CSS）での銀行振込経路。システム上は事務局が手動で入金・会員情報を更新する想定。**1月の自動カード請求は行わない**。
- 会員編集のチェックとマイページの「クレジットに切り替える」でフラグを管理。

## 2. ICA会員フラグ — 実装済み

- **DB**: `008_ica_export_css_user.sql` で `ica_requested`, `ica_exported`, `ica_exported_at`, `is_css_user` を追加（`ica_exported*` は旧ワークフロー用。一覧・絞り込みは **`is_ica_member` のみ**）。
- **入会**: Stripe メタデータで ICA 希望の場合、Webhook で **`is_ica_member` と `ica_requested` を同じ値**で作成する（ICA会員として登録）。
- **会員一覧**: ICA列は `is_ica_member` のみ（○／－）。ICA専用のCSV出力 API・画面ボタンは廃止。
- 会員編集フォームに「ICA会員」「CSS（口座振替）対象」チェックを追加済み。

## 3. 26歳ルール — 実装済み

- **学生会員→正会員の自動昇格**: `GET /api/cron/student-to-regular`。`birth_date` から「26歳に達した年」を算出し、`membership_type` を `student` → `regular` に一括更新。実行は年1回（例: 1月16日）を想定。`vercel.json` の crons に登録済み。要 `CRON_SECRET`（Authorization: Bearer または x-cron-secret）。
- **12月予告メール**: `GET /api/cron/student-upgrade-preview-email`。来年度26歳になる学生会員（現在25歳）に「来年度から正会員になります」メールを送信。12月1日実行想定。同上で `CRON_SECRET` 必須。

## 4. 入会フォームとマイページ — 実装済み

- **入会フォーム**: `/membership/join` → `POST /api/membership/checkout` → Stripe Checkout → Webhook で profile + membership 作成。完了ページは `/membership/join/complete`。
- **マイページ「クレジットカード支払いに切り替える」**: `POST /api/mypage/switch-to-card` で `is_css_user = false`。

## 5. 自動メール送信 — 実装済み

- **入会完了・決済成功時**: Stripe Webhook で本人・事務局へメール。

### メールが届かないときの確認

**補足：Supabase の Security Advisor（RLS 無効・ポリシー警告など）はメール送信とは無関係です。** メールは Vercel 上の Next.js（nodemailer）で送っており、Supabase のトリガーや RLS は使っていません。

#### チェックリスト（本番で届かない場合）

1. **メール送信だけ試す（原因切り分け）**
   - 管理者でログインした状態で、ブラウザで **`https://jca2026hp.vercel.app/api/admin/email-test`** を開く。
   - 「テストメールを ○○ に送信しました」と表示され、そのアドレスにメールが届けば、**メール設定（EMAIL_USER / EMAIL_APP_PASSWORD）は問題なし**。届かない場合は「送信に失敗しました」や「EMAIL_USER が未設定」などが表示されるので、Vercel の環境変数を確認する。

2. **Stripe Webhook が呼ばれているか**
   - Stripe ダッシュボード → **開発者 → Webhook** → 登録したエンドポイントを開く。
   - **「最近のイベント」**（または「送信の履歴」）で、入会決済後に `checkout.session.completed` が **成功（200）** で送られているか確認する。
   - **失敗（4xx/5xx）** や **送信されていない** 場合は、Webhook URL や `STRIPE_WEBHOOK_SECRET` の設定ミス、または本番のエラーが原因。

3. **Vercel の環境変数**
   - Vercel → プロジェクト → **Settings → Environment Variables** で、**Production** に以下が入っているか確認する。
   - `EMAIL_USER`（Gmail アドレス）、`EMAIL_APP_PASSWORD`（Gmail アプリパスワード）、`STRIPE_WEBHOOK_SECRET`（Stripe の署名シークレット `whsec_...`）。
   - **1月自動請求 Cron**: `CRON_SECRET` を設定し、Vercel Cron が `/api/cron/membership-january-charge` を呼べるようにする。
   - 変更したあとは **Redeploy** が必要。

4. **Gmail で送信元が拒否される場合**
   - `EMAIL_FROM="日本クラリネット協会 <あなたの@gmail.com>"` を Vercel の環境変数に追加し、EMAIL_USER と同じアドレスにする。

5. **本番で Webhook を動かす手順**
   - 詳しくは **[Stripe Webhook 本番設定の手順.md](./Stripe%20Webhook%20本番設定の手順.md)** を参照。
