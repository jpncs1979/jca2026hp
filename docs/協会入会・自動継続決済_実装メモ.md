# 協会入会・自動継続決済システム 実装メモ

## 1. 決済・更新ロジック (Stripe) — 入会時は実装済み

- **入会時決済**: `POST /api/membership/checkout` で入会フォーム送信 → Stripe Checkout 作成。`src/lib/membership-fees.ts` で入会金2,000円・正会員8,000円・学生会員6,000円を定義。**10〜12月入会（案B）**: 初回のみ「入会金＋当該年度会費＋翌年度会費」を `getMembershipJoinAmount()` で合算して Checkout に渡している。決済完了は Webhook `checkout.session.completed`（metadata.type=membership_join）で profile・memberships・payments 作成し、入会完了メールを送信。
- **毎年1月15日の自動継続決済**: 未実装。Stripe Subscription の年1回課金または Vercel Cron での一括インボイス作成が必要。

## 2. ICA連携とCSV出力 — 実装済み

- **DB**: `008_ica_export_css_user.sql` で `ica_requested`, `ica_exported`, `ica_exported_at`, `is_css_user` を追加。
- **新規ICA希望者出力**: 会員管理画面の「新規ICA希望者出力」ボタンで、`ica_requested = true` かつ `ica_exported = false` の会員をCSV出力し、出力後に `ica_exported = true`, `ica_exported_at = now()` に更新。
- **ICA再出力（期間指定）**: 「ICA再出力（期間指定）」ボタンでダイアログを開き、開始日・終了日を指定。`ica_exported_at` または入会日が範囲内のICA希望者をCSV出力（フラグは更新しない）。
- **API**: `GET /api/admin/members/ica-export?mode=new` および `?mode=re&from=YYYY-MM-DD&to=YYYY-MM-DD`。
- 会員編集フォームに「ICA希望」「CSS（口座振替）対象」チェックを追加済み。

## 3. 26歳ルール — 実装済み

- **学生会員→正会員の自動昇格**: `GET /api/cron/student-to-regular`。`birth_date` から「26歳に達した年」を算出し、`membership_type` を `student` → `regular` に一括更新。実行は年1回（例: 1月16日）を想定。`vercel.json` の crons に登録済み。要 `CRON_SECRET`（Authorization: Bearer または x-cron-secret）。
- **12月予告メール**: `GET /api/cron/student-upgrade-preview-email`。来年度26歳になる学生会員（現在25歳）に「来年度から正会員になります」メールを送信。12月1日実行想定。同上で `CRON_SECRET` 必須。

## 4. 入会フォームとマイページ — 実装済み

- **入会フォーム**: `/membership/join` で氏名（漢字・カナ）、生年月日、性別、住所、電話、メール、所属（一般/プロ/学生）、会員種別（正会員/学生会員）、ICA希望を入力。送信で `POST /api/membership/checkout` → Stripe Checkout（案B対応）→ 決済完了で Webhook が profile + membership 作成。完了ページは `/membership/join/complete`。
- **マイページ「クレジットカード支払いに切り替える」**: 
  - API `POST /api/mypage/switch-to-card` を用意済み（要実装の場合は「カード登録完了後」にこのAPIを叩くか、Stripe Webhook で `is_css_user = false` に更新）。
  - マイページにボタンを設置し、Stripe Customer Portal または Checkout へ遷移。戻り後に上記APIを呼んで `is_css_user` を `false` に更新する流れを想定。

## 5. 自動メール送信 — 実装済み

- **事務局承認時**: 既存の `approve` API で入会承認メール送信あり。
- **入会完了・決済成功時**: Stripe Webhook `checkout.session.completed`（metadata.type=membership_join）で profile 作成後に「ご入会手続きが完了しました」メール（本人）と「【事務局】新規入会の通知」（事務局）を送信。

### メールが届かないときの確認

1. **入会完了メールは Webhook で送信**
   - ローカル開発では Stripe から Webhook が呼ばれません。メールを試すには (a) 本番（Vercel）にデプロイして Stripe の Webhook URL を本番に設定する、(b) または [Stripe CLI](https://stripe.com/docs/stripe-cli) で `stripe listen --forward-to localhost:3000/api/webhooks/stripe` を実行し、表示される `whsec_...` を `.env.local` の `STRIPE_WEBHOOK_SECRET` に設定してから決済テストする。

2. **環境変数**
   - `EMAIL_USER` と `EMAIL_APP_PASSWORD`（Gmail のアプリパスワード）が設定されているか。
   - `STRIPE_WEBHOOK_SECRET` が空だと Webhook 全体が 400 で拒否され、メール処理まで到達しません。

3. **Gmail で送信元が拒否される場合**
   - Gmail は「認証したアカウントと異なる From」を拒否することがあります。`.env.local` に次を追加して、送信元を Gmail アドレスに揃えてください。
   - `EMAIL_FROM="日本クラリネット協会 <jpncs1979@gmail.com>"`（EMAIL_USER と同じアドレスを推奨）

4. **ログ**
   - ターミナルに `[入会完了メール送信エラー]` や `[事務局通知メール送信エラー]` が出ていないか確認。出ている場合はその後のメッセージで原因が分かります。

5. **本番で Webhook を動かす手順**
   - 詳しくは **[Stripe Webhook 本番設定の手順.md](./Stripe%20Webhook%20本番設定の手順.md)** を参照（Vercel デプロイ後、Stripe で Webhook URL を本番の `/api/webhooks/stripe` に設定する手順を記載）。
