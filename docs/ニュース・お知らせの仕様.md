# ニュース・お知らせの仕様

## ページ構成（`/news`）

1. **お知らせ** … 協会からの告知を一覧表示
   - **常時表示**：ICA入会のお知らせ（「今なら入会するとICA会員にもなれます」）を先頭に固定表示。詳細・入会へのリンク付き。
   - **DB連携**：Supabase の `news` テーブルから取得した記事を、公開日順で最大20件表示。
2. **会員後援演奏会** … チラシの横スクロール・カレンダー・一覧へのリンク

---

## お知らせ（DB連携部分）の仕様

### データソース

- **Supabase** のテーブル名：`news`
- 管理画面は本プロジェクトには含まれていません。Supabase ダッシュボード（Table Editor）または API で記事の追加・編集を行います。

### カラム（想定）

| カラム名       | 型        | 説明 |
|----------------|-----------|------|
| `id`           | uuid 等   | 一意ID（主キー） |
| `title`        | text      | タイトル |
| `content`      | text      | 本文（改行はそのまま表示） |
| `publish_date` | date      | 公開日（YYYY-MM-DD）。この日付以前の日付の記事のみ表示 |
| `is_important` | boolean   | 重要フラグ。true のとき「重要」バッジ表示・カードの枠をゴールドで強調 |

### 表示ルール

- **公開日**：`publish_date <= 今日` の記事のみ表示（未来日付は非表示）
- **並び順**：`publish_date` の降順（新しい順）
- **件数**：最大 20 件
- **表示**：1記事1カード。日付・重要バッジ（任意）・タイトル・本文。本文は `whitespace-pre-wrap` で改行を反映。

### 個別記事ページ

- 現状、`/news/[id]` の個別記事ページは未実装です。トップページの「トピックス」はモックデータを参照しており、リンク先は未実装の可能性があります。必要に応じて `app/news/[id]/page.tsx` を追加し、`news` テーブルから1件取得して表示する実装が考えられます。

---

## 運用メモ

- ICAのお知らせカードはコードに直書きしているため、非表示にしたい場合は `src/app/news/page.tsx` の該当ブロックを削除またはコメントアウトしてください。
- お知らせの新規追加・更新は Supabase の `news` テーブルに対して行ってください。
