## 1. やることの全体像

このセットアップでは、**2つの作業**を行います。

| 作業 | 内容 | 所要時間の目安 |
|------|------|----------------|
| **手順A** | 環境変数の設定 | 約5分 |
| **手順B** | データベースにテーブルを作成 | 約3分 |

両方完了すれば、サイトから申込フォームやお知らせ機能が使えるようになります。

---

## 2. 手順A：環境変数の設定

サイトがSupabaseのデータベースに接続するために、**プロジェクトのURL**と**鍵（キー）**を設定します。

### ステップ1：Supabaseにログインする

1. ブラウザで [https://supabase.com](https://supabase.com) を開く
2. 右上の **「Sign in」** をクリック
3. アカウントでログインする（なければ新規登録）

### ステップ2：プロジェクトを開く

1. ダッシュボードに、作成済みのプロジェクト一覧が表示されます
2. **日本クラリネット協会**のプロジェクトをクリックして開く
3. ※ プロジェクトがまだない場合は、先に開発者に作成を依頼してください

### ステップ3：URLと鍵を取得する

1. 左側メニューの **「Project Settings」**（歯車アイコン）をクリック
2. 左側の **「API」** をクリック
3. 画面に表示される以下の2つをメモまたはコピーする

| 項目 | 場所 | 例 |
|------|------|-----|
| **Project URL** | 「Project URL」と書かれた欄 | `https://abcdefgh.supabase.co` |
| **anon public** | 「Project API keys」の一覧にある「anon」「public」の値 | `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`（長い文字列） |

> ⚠️ **注意**：anon key（anon public）は他人に見られないように保管してください。GitHubなどに公開しないでください。

### ステップ4：.env.local ファイルを作成する

1. プロジェクトのフォルダを開く（`2026clHP` フォルダ）
2. **「.env.local」** という名前のファイルを新規作成する
   - Windowsの場合：メモ帳で作成し、「すべてのファイル」で保存して `.env.local` と名前をつける
   - すでに `.env.local` がある場合は、そのファイルを開く
3. 以下の内容を**自分の値に置き換えて**書き込む

```
NEXT_PUBLIC_SUPABASE_URL=ここにProject URLを貼る
NEXT_PUBLIC_SUPABASE_ANON_KEY=ここにanon publicの鍵を貼る
```

**記入例**

```
NEXT_PUBLIC_SUPABASE_URL=https://abcdefgh.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZSI...
```

> ⚠️ **ポイント**
> - `=` の前後にスペースを入れない
> - 値の前後に余計な引用符（`"` や `'`）を付けない
> - 1行で書く（途中で改行しない）

### ステップ5：開発サーバーを再起動する

1. ターミナルで `npm run dev` が動いている場合は、**Ctrl + C** で止める
2. もう一度 **`npm run dev`** を実行する

これで環境変数の設定は完了です。

## 3. 手順B：データベースにテーブルを作成する

申込データやお知らせを保存するための「テーブル」をデータベースに作成します。

### 方法1：Supabaseダッシュボードから実行（おすすめ）

ダッシュボードのSQL画面で、用意されたSQLを貼り付けて実行する方法です。**こちらを推奨**します。

#### 手順

1. ブラウザでSupabaseのプロジェクトを開く（手順Aのステップ2までと同じ）
2. 左側メニューの **「SQL Editor」** をクリック
3. **「New query」** をクリックして、新しいクエリ用の入力欄を開く
4. プロジェクト内の **`supabase/migrations/001_create_tables.sql`** を開く
   - エディタ（VS CodeやCursorなど）で開く
   - ファイルの中身を**すべて選択**（Ctrl + A）して**コピー**（Ctrl + C）
5. SupabaseのSQLエディタに **貼り付け**（Ctrl + V）
6. **「Run」** ボタンをクリック（またはキーボードで Ctrl + Enter）
7. 画面下部に **「Success. No rows returned」** と表示されれば成功です

8. **`supabase/migrations/002_add_email_to_applications.sql`** も同様に開き、内容をコピーして SQL エディタに貼り付け、**「Run」** で実行する  
   - 申込フォームのメールアドレス保存・メール送信機能に必要です

#### 確認

1. 左側メニューの **「Table Editor」** をクリック
2. `competitions`、`applications`、`news` の3つのテーブルが表示されていればOKです
3. `applications` テーブルに `email` カラムが含まれていればOKです（002 実行後）

### 方法2：Supabase CLI を使う（上級者向け）

コマンドラインから一括で実行する方法です。開発者向けです。

#### 前提

- Supabase CLI がインストールされている
- `supabase login` でログイン済み
- プロジェクトがSupabaseとリンク済み（`supabase link` 済み）

#### 手順

1. ターミナルでプロジェクトフォルダ（`2026clHP`）に移動
2. 以下を実行

```
supabase db push
```

3. エラーが出ずに完了すればOKです

## 4. 動作確認

1. ブラウザで [http://localhost:3000](http://localhost:3000) を開く
2. サイトが表示されることを確認する
3. 申込フォームやお知らせページなど、データベースを使う機能がエラーなく動作すればセットアップ完了です

## 5. うまくいかない場合

### 環境変数が反映されない

- `.env.local` を保存したあと、**必ず `npm run dev` を再起動**してください
- ファイル名が `.env.local` になっているか確認（`.env.local.txt` などになっていないか）
- `.env.local` はプロジェクトの**一番上のフォルダ**（`2026clHP`）に置く

### SQL実行でエラーが出る

- `001_create_tables.sql` の内容を**最初から最後まで**すべてコピーしているか確認
- すでに同じテーブルが存在する場合は、「Table already exists」などのエラーが出ることがあります。その場合は開発者に相談してください

### プロジェクトがまだない

- Supabaseで新しいプロジェクトを作成する必要があります
- 作成後、手順Aのステップ3からやり直してください
