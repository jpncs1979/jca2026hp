# 新規コンクール作成手順書

新しいコンクール（例：第16回ヤングコンクール、アンサンブルコンクールなど）をシステムに追加するための手順です。技術的な知識がなくても、この手順に沿って進めれば作成できます。

## 1. 事前に準備するもの

新しいコンクールを登録する前に、以下の情報を用意してください。

| 項目 | 説明 | 例 |
|------|------|-----|
| **コンクール名** | 正式名称 | 第16回ヤングコンクール |
| **開催年** | 西暦4桁 | 2027 |
| **年齢基準日** | 年齢を判定する基準の日付（通常は4月1日） | 2027年4月1日 |
| **カテゴリ一覧** | 参加者が選べる部門 | ジュニアA、ジュニアB、ヤング |

### カテゴリについて

- 参加者が申込時に選択する「部門」です
- カンマや「、」で区切って一覧にします
- 例：`ジュニアA, ジュニアB, ヤング` または `アンサンブル部門, ソロ部門`

## 2. Supabaseダッシュボードにログインする

1. ブラウザで [Supabase](https://supabase.com) を開く
2. 右上の **「Sign in」** をクリックしてログインする
3. 対象プロジェクト（日本クラリネット協会のプロジェクト）を選択する
4. 左側メニューの **「SQL Editor」** をクリックする

> 左側メニューの下の方にある「SQL Editor」（データベースのアイコン）をクリックします。

## 3. 新しいコンクールをデータベースに登録する

### ステップ1：SQLエディタを開く

1. SQL Editorの画面で **「New query」** ボタンをクリック
2. 新しいクエリ用の入力欄が表示されます

### ステップ2：登録用のSQLを入力する

以下のテンプレートをコピーし、**ご自分のコンクールの内容に書き換えて**から貼り付けてください。

```sql
INSERT INTO competitions (slug, name, year, reference_date, category_options) 
VALUES (
  'ここに英数字の識別子',
  'ここにコンクール名',
  開催年を数字で,
  '基準日の日付',
  '["カテゴリ1", "カテゴリ2", "カテゴリ3"]'::JSONB
);
```

### ステップ3：各項目を書き換える

#### ① slug（識別子）

- **用途**：システム内部でコンクールを区別するためのID
- **ルール**：半角英数字とハイフン「-」のみ使用。空白・日本語は使わない
- **書き換え例**：
  - 第16回ヤングコンクール 2027 → `young-2027`
  - アンサンブルコンクール 2027 → `ensemble-2027`

#### ② name（コンクール名）

- **用途**：画面に表示される正式名称
- **書き換え例**：`'第16回ヤングコンクール'`

#### ③ year（開催年）

- **用途**：開催年
- **書き換え例**：`2027`（数字のみ、引用符なし）

#### ④ reference_date（年齢基準日）

- **用途**：参加者の年齢を判定する基準日（通常は4月1日）
- **形式**：`'YYYY-MM-DD'`（年-月-日）
- **書き換え例**：`'2027-04-01'`

#### ⑤ category_options（カテゴリ一覧）

- **用途**：参加者が選べる部門の一覧
- **形式**：`'["部門1", "部門2", "部門3"]'`
- **ポイント**：
  - 各部門名はダブルクォート「"」で囲む
  - 部門どうしはカンマ「,」で区切る
  - 全体を角括弧「[ ]」で囲む
  - 最後に `::JSONB` を付ける（そのままコピーしてOK）

- **書き換え例**：
  - ヤングコンクール：`'["ジュニアA", "ジュニアB", "ヤング"]'::JSONB`
  - 2部門のみ：`'["ソロ部門", "アンサンブル部門"]'::JSONB`

### ステップ4：完成例

第16回ヤングコンクール（2027年開催）の場合の完成例です。

```sql
INSERT INTO competitions (slug, name, year, reference_date, category_options) 
VALUES (
  'young-2027',
  '第16回ヤングコンクール',
  2027,
  '2027-04-01',
  '["ジュニアA", "ジュニアB", "ヤング"]'::JSONB
);
```

### ステップ5：SQLを実行する

1. **「Run」** ボタン（または Ctrl + Enter）を押す
2. 画面下部に **「Success. No rows returned」** と表示されれば成功です
3. エラーが出た場合は、[よくあるエラーと対処法](#よくあるエラーと対処法) を確認してください

## 4. 登録内容の確認

登録が正しくできているか確認する方法です。

### 確認用SQLの実行

1. SQL Editorで **「New query」** をクリック
2. 以下のSQLを貼り付けて実行する

```sql
SELECT * FROM competitions ORDER BY year DESC, created_at DESC;
```

3. 一覧に新しいコンクールが表示されていれば登録完了です

## 5. よくある質問

### 同じコンクールを2回登録してしまった場合は？

`slug` が重複しているとエラーになります。重複登録された場合は、開発者に連絡して削除してもらってください。

### コンクール名を間違えて登録した場合は？

登録後は、開発者がSQLで修正する必要があります。修正したい場合は担当者に連絡してください。

### 申込受付をいったん停止したい場合は？

`is_active` を `false` にする必要があります。こちらもSQLでの更新が必要なため、開発者に依頼してください。

### カテゴリを後から追加・変更したい場合は？

`category_options` の変更はSQLでの更新が必要です。開発者に依頼してください。

## 付録：よくあるエラーと対処法

### エラー：`duplicate key value violates unique constraint`

**原因**：同じ `slug` のコンクールがすでに登録されている  
**対処**：`slug` を別の値に変更する（例：`young-2027` → `young-2027-v2`）

### エラー：`invalid input syntax for type json`

**原因**：`category_options` の形式が正しくない  
**対処**：以下を確認する
- 各部門名が `"` で囲まれているか
- カンマの位置が正しいか
- 最後に `::JSONB` が付いているか
- 全体が `'`（シングルクォート）で囲まれているか

### エラー：`syntax error at or near ...`

**原因**：引用符やカンマの使い方が誤っている  
**対処**：
- 日本語の項目（name、category_options内）は `'` で囲む
- 数字（year）は引用符をつけない
- カンマの後にスペースを1つ入れる
